前言：于 2018 年本科毕业自某双非一本，相对行业是个菜鸟。目前和未来的职业规划都是 Android 开发工程师，有自学一点浅显的机器学习——推荐系统，以及 Python、C/C++。本文将一些个人认为比较重要、或是比较有参考价值的面试，尽量详细地记录下来，但仅记录面试问题，解答另外记录在面试知识体系内：常见面试题知识体系汇总。

腾讯

  • 应聘部门：微视商业化

  • 应聘岗位：Android 开发工程师

  • 邀约时间：2020-03-05

  • 面试结果：等待面委会最后决定。

整体感受：腾讯是我个人目前面试经历中感受最好的一家，腾讯毕竟有微信作为载体，面试安排、交流反馈等等都充分利用了微信的平台，面试时间只要还算合理，一般都能自己安排，HR 小姐姐也很亲和，交流得比较丰富。有个小插曲，在面腾讯之前面过一家国内某 T2 级别的大厂，因为接到面试邀约的电话很仓促，所以和 HR 约定三天后面试，结果第二天面试官打电话过来，说后面还有很多面试要安排，不要等那么久就这两天。所以在接到腾讯电话后，我本能地约在了第二天下午（03-06，面试官也确认同意了），后来 HR 来联系询问相关安排后，特意提醒我怎么这么急，有没有复习好，要不要推迟几天，于是赶紧往后又延长了 3 天（03-09），事实也证明 HR 的建议是很正确的。面试官的水平都很高（也可能是我的水平太低），参与面试就很享受，有很多问题就像面试官在引导我去参与、思考，面试完就像上了一节课一样。

一面（技术面）

一面面试日期：2020-03-09，总时长约 80 分钟。

一面体会：BAT 不愧是大厂，参与面试就是一个很享受的过程，一面的面试官应该是该岗位入职后的同事级别，面的过程还是比较注重细节的，有很多问题都会尝试深入去问，能感觉得出来面试官也是很认真的准备了面试。面完能明显感觉到自己的短板，本来也觉得希望不大，没想到还是过了，收到通过的消息还是很激动的。

（1）编程题

  • 冒泡排序

  • 二叉树层序遍历（广度优先遍历）

（2）技术题

  • 设计模式（工厂、单例、中介等）

  • 应用架构（MVC、MVP、MVVM）

  • Handler 原理、源码

  • 进程间通信

  • Bundle 的原理

  • Activity 使用 Handler 的风险（内存溢出）

  • HTTP 和 HTTPS

  • TCP 的三次握手和四次挥手

  • 热补丁和插件化（原理、理解）

  • ClassLoader 的原理、ClassLoader 如何查找类

  • View 的绘制流程

  • HashMap 原理、源码

  • Socket

  • Java 中引用的实现

  • GC 算法

（3）开放题

  • 工作以来最有挑战性的事

  • 自己设计一个文件下载器

二面（技术面）

二面面试日期：2020-03-14，总时长约 66 分钟。

二面体会：二面基本上都是在针对简历中写到的点在问，问的很仔细，基本上项目经历中写到的每句话都问了相关知识点，还好自己的简历写得还是比较真实。我的简历中甚至还出现了一个错误，结果在过去几个月的面试中都没有发现，这一次被指出来也是相当尴尬，总结出三点：① 简历可以包装但绝对不能过度吹嘘；② 项目经历中涉及到的技术点，或者说做项目时遇到的技术点，一定要尝试去深入了解、知其所以然，不然很容易一问就倒；③ 面试过程一定要真诚，会就会、不会就不会，可以委婉表示或者尝试切入到自己会的内容，但切忌不懂装懂。

（1）编程题

二面没有考编程题，也没有手写算法。

（2）技术题

  • （针对项目）介绍一下这个项目（简历中写了三个，都问到了）

  • （针对项目）为什么选择 Socket 进行数据传输？

  • （针对项目）申请了什么权限？如果用户没授权怎么办？

  • （针对项目）你都做了哪些用户体验的优化？

  • （针对项目）数据的缓存是怎么实现的？Glide 的缓存（项目中用的是 LRUCache）有了解过吗？

  • （针对项目）有没有做过一些性能指标的监控或者优化，比如启动速度、帧率等等？

  • （针对项目）数据传输的速度有多快？有没有导致 App 卡顿？如果数据量过大，有没有做一些优化（例如降采样等）？

  • （针对项目）用的数据协议是什么？方便透露一下协议的格式吗？

  • （针对项目）做了哪些数据埋点？

  • （针对项目）方便透露一下用户量吗？

  • （针对项目）如何优化的 RecyclerView 中的视频 Item？

  • （针对项目）多久更新一个版本？

  • （针对项目）异常上报和热更新（项目里用的是 Bugly 和 Tinker）是怎么实现的？Tinker 的原理有了解过吗？

  • （针对项目）项目中 FFmpeg 是怎么用的？

  • LRU 算法的理解

  • 在收到异常上报时，一般因为混淆过，可能收到的报告看不出来实际的地方，那你是如何解决的呢？

  • 项目中哪些情况下会出现 ANR，如果遇到 ANR 如何定位和修复呢？

  • 什么情况下会发生 OOM？如何定位和优化？

  • 有哪些方法可以监测 FPS？

  • LeakCanary 的原理有了解过吗？

  • 简述一下进行 JNI 开发的流程

  • 如果发生了 Native Crash，如何定位和修复？

  • 平时提交代码时，谁来 Review 以及如何 Review？

  • 有做过单元测试吗？是自己写得还是直接用的自带的？

  • Python 中的 nosetest 的原理？

  • 有没有遇到过比较难解决的报错或者 Crash（Application 不在主 dex 导致混淆后报 ClassNotFound）？最后又是怎么解决的？

（3）开放题

  • 自己去设计一个线程池的话，会怎么考虑，例如怎么管理线程等

  • 平时写代码遵循的是：谷歌、阿里、或者是你自己的一套代码规范？

  • 平时玩游戏吗？假如你在休假时间在玩游戏，这时候领导打电话说项目出问题了，你会怎么解决？假如这个问题不是你负责的呢？假如你身边没有条件（例如没有电脑等）立即解决，又会如何处理？

三面（交叉总监技术面）

三面面试日期：2020-03-19，总时长约 60 分钟。

三面体会：三面是另一个部门的总监交叉面，气场很强大，面试过程虽然没有故意刁难我，也没有咄咄逼人，但是严肃的氛围还是让我压力山大（笑）。在简历中有写到在校期间学过 C/C++，以及项目中用到了简单的 JNI 开发，但是在一面之前，HR 小姐姐有跟我说这个岗位比较基础，目前没有音视频这块的经验也是 OK 的，所以我也就心大的没有准备这块的内容，然后面试官特意问了几个这块的内容，结果我太久没接触了所以没答好……当然，这绝对是我自己的锅，还是那句话，简历上写的东西，一定要是琢磨透了的，没把握的一定不要往上写，有时候自己觉得什么都懂一点点是加分项，其实到了境界更高技术更厉害的人眼里，反倒是减分项，务必自勉，不可浮躁。

（1）编程题

  • 手写最安全的单例模式。

    @注：我个人觉得，在不考虑性能的情况下，双锁检测和静态内部类的安全性是一致的，所以两个都写了，如果考虑性能，静态内部类会更好一些。

（2）技术题

  • （针对项目）介绍一下这个项目（简历中的三个项目问了两个）

  • 介绍一下除了工厂和单例以外的设计模式（选了观察者模式和中介者模式）

  • 观察者模式如何保证线程安全？如果加锁（例如多个线程同时取消注册），如何防止死锁？

  • 死锁的产生条件以及如何解除？

  • ClassLoader 的原理，以及为何要使用双亲委托制？

  • equals 和 == 的区别

  • 为什么要有 hashcode() 方法？

  • Handler 的原理

  • ANR 和 OOM 的原因、监测、和修复

  • 做过哪些启动优化和 APK 体积优化？

  • 针对 RecyclerView 做过哪些优化？

  • 数组和指针的区别？

  • 介绍一下虚函数？（因为简历上写学过 C/C++，项目中用到了 JNI，但是太久没接触了没答出来）

  • View 的绘制流程

  • JVM 内存模型和分区

  • Java 的线程同步机制

  • Java 中有哪些锁？以及 synchronized、volatile、和 lock 有什么区别？

  • UI 主线程中的消息处理机制是怎样的？如果也是 Looper 循环，不会导致死循环吗？

  • 在浏览器中输入 www.qq.com 或者 www.baidu.com，之后会发生什么事？

（3）开放题

这一轮没有问开放题，基本都是宏观一些的技术问题。

四面（HR面）

四面面试日期：2020-03-19，总时长约 30 分钟。

四面体会：三面面完之后本来也是感觉悬，结果半个小时不到，HR 小姐姐就专程打电话过来通知我三面过了，还问我接到电话通知是不是更高兴（哈哈，小姐姐有心了，赞！），然后就约了当天晚一点的 HR 面，不过 HR 面的面试官是另一位来自微视 HR 部门的，面试过程，虽然比起技术面比较轻松了，但问题还是比较“直击灵魂”，感觉面试官是个很有经验的 HR，所以建议 HR 面的时候，还是要诚实守信，真诚一点。我自己感觉是真的热爱这个行业也花了功夫，所以回答都是实话实说，对得起自己的良心最重要（油腻比心）。另外，这一轮只问了我上一家公司的薪资情况，没有聊我的预期薪资，可能是因为我面的是基础岗（肯定是 T2.3 以下，不然还要有个通道面），所以直接就是先定级然后按级定薪。

（1）编程题

无

（2）技术题

无

  • 有用过微视吗？对于这款产品，你觉得哪里还可以改进？（一上来就一道杀招，还真没想过会有这个类型的问题，其实主要目的应该是考察有没有去了解过自己应聘的岗位吧）

  • 在过去的工作经验中，有没有遇到什么比较大的困难？最后又是如何解决的呢？

  • 你对加班是怎么看的？

  • 在业余时间都会做什么呢？

  • 在业余时间里提升自己的时候都用到了哪些方法呢？

  • 有什么和技术无关的爱好吗？

  • 家庭情况怎么样，有没有女朋友？

  • 你对自己未来有没有一个很清晰很明确的规划？

华为

  • 应聘部门：消费者 BG 音频产品开发部

  • 应聘岗位：Android 开发工程师

  • 邀约时间：2020-02-25

  • 面试结果：已通过面试和提交材料，等待 Offer 发放流程。

整体感受：面试过程不好评论，面完第一轮才知道是“高端外包”，不过既然已经面了一次，就索性坚持到底了。就像匆匆忙忙的面了，又匆匆忙忙的过了（orz）。不过说实话交流效率还是有待提升，和 HR 联系时常收不到回复，可能是工作忙，也可以理解。测试的很全面，机试题、性格测试、面试都有。机试题直接用的牛客网做的，电脑当前页面监控以及摄像头抓拍，还需要手机打开一个小程序用来监测没有切换页面去搜答案等，不过难度都不大，十道单选多选，两道编程题，基本上十来分钟做完。面试难度也不大，面试过程和节奏也都比较轻松，也可能因为是外包的原因，所以整体难度都下调了一些。

@注：其实面试的是华为的高端外包 OD，据 HR 和面试官说，华为现在只有职级在 18 以上的高端岗位才社招，13-17 级的岗位不社招了，都是通过校招、OD、以及普通外包这三种方式招人。根据 HR、面试官、某位华为正式员工朋友的说明以及在网上收集到的信息来看，OD 和普通外包的一些信息和区别如下：

普通外包就是类似于：中软、软通、佰钧成、东软、文思海辉等，合同一般一年一签，人事制度由外包公司管理，有时工作地点不在华为，不接触核心业务，工号 WX（外协）开头，员工权限和工资低于正式员工，无奖金、无期权。

OD 和德科签合同，四年一签，人事制度由华为管理，工作地点在华为，接触核心业务，工号 3 开头（专门开设的），员工权限、工资、奖金和正式员工一致（据说实际上还是比正式员工差，也可以理解），但没有期权，要求学历一本及以上，每年有 20% 的机会可以转为正式员工。

一面体会：有两个面试官，都挺随和的，问题比较基础，有些不太清楚的也会给些提示引导我去思考，整体氛围比较 Nice，不会感到紧张。手写程序题是面完结束后才写的，然后拍照邮件发给 HR 再审核。

（2）技术题

（3）开放题

没有问开放题，基本都是一问一答的形式。

数码视讯

  • 应聘部门：深圳研究院一所

  • 应聘岗位：Android 开发工程师

  • 邀约时间：2019-11-07

  • 面试结果：拿到 Offer

整体感受：面试过程能感觉到面试官的诚意，第一次有被重视和珍惜“人才”的感觉，初面其实并不顺利，但二面三面和面试官都很聊得来，算是重新挽回甚至逆袭了吧。不过比较遗憾的是，正式入职一天就辞职了，入职后才发现和一开始了解到的偏差很大，有些东西一开始没有互相了解清楚，最终耽误了双方的时间，结局不算愉快。其实拿到 Offer 后，入职体检、体检了解、预习项目等等都认真去做了，是真心抱着要去发展的心态入职的，连周边的房租都了解了一圈，公司为了入职的流程也做了很多工作，HR 也帮我申请了一次加薪机会，而且也确实批下来了，虽然试用期期间离职也是我的合法权利，但整体而言，还是比较感激这家公司给我的机会和诚意的，也确实对最后没能愉快合作感到抱歉。不过话说回来，做技术这行，我个人还是认为，选择大于努力，如果发现道路、方向不对，一定要有决心朝着自己的目标走，毕竟有时候一个选择的失误，可能会导致很多不可挽回的后果。

  • 邀约时间：2019-10-12

  • 面试结果：一面挂 + 二面挂

整体感受：由于毕业后急于找工作，没有好好准备就入职了第一家公司，所以对于面试经验积累得比较晚，简历也是后来才完善起来，所以在计划离职开始找工作的开始一段时间，收到的邀请都是比较差的，也一直没有拿到心仪的公司的邀请函。后来一次机会接到了白骑士的邀请前去面试，也 曾经 成为了我参与面试的公司里面最心仪的一家，甚至愿意降薪加入，不过很遗憾最终没有拿到 Offer。值得一提的是，这家公司我参与了两次面试，第一次是一面挂，第二次是二面挂。第一次面完一面后，面试官对我的评价挺高的（恕我无能，真的没听出来有反讽或者客套或者安慰的意思，是真的感觉到在赞同我），但是等了一周都没有消息，期间打前台电话咨询了两次，两次都说帮我登记了，稍后回电告知，但直到现在我也没等到那个“稍后”的回电。一个星期后我觉得就是婉拒了吧，结果又接到另一位 HR 的邀约，我还是没放弃，猜测可能是第一次提的期望薪资太高了，所以又带着一丝希望前往面试，这次期望薪资少说了 2K（直接说的岗位邀约薪资范围的最低数），笔试和一面还是很顺利的通过了，二面过程也比较顺利，甚至都过到了 HR 终审的环节，结果回去以后，又是杳无音讯，期间打过一次电话询问结果，和第一次一样，还是到现在都没等到那通回电。我个人是很反感这种行为的，如果技术不过关，或者期望薪资太高，可以直接和我说，如果是让回去等通知婉拒，也没必要都主动打电话问了还这么敷衍。或许是我水平太差，入不了白骑士的法眼，但对于这种行为，我送上个人角度的差评。

同行者科技

  • 应聘部门：应用开发部

  • 应聘岗位：Android 开发工程师

  • 邀约时间：2019-10-08

  • 面试结果：二面挂

整体感受：这家公司我是二面挂的，但其实我并没有讨厌或者怀恨的心态，反而可以说是我后来面试的一个转折点，所以其实我自己还是蛮感激的。去面试报到后，同行的应聘人是相当多，光是 Android 都同时有好几位竞争者，每个人都发了一份 Java 编程题，一共三道，让一个小时内做完，现在回想题目并不难，不过由于当时刚计划离职开始面试，没有准备也没有经验，所以做得其实不咋样，而且被安排在一个桌子上写得还有另一个年纪相仿的同行，一直在偷看我的答案，明着挡暗着挡好几回了也不收敛，虽然我也做的不咋样，但还是相当反感这种作弊行为，后来做完等了一会被叫进去面试，先是一面的面试官，直接对笔试题，因为当时水平是真的不行，有很多边界值、限定范围等等的没有考虑到，还有一些小的逻辑错误，所以批改的惨不忍睹，不过一面的面试官很有耐心，一直在和我分析、带我反思，批改完也没问别的，让我好好改改，然后去叫二面的面试官。二面面试官进来后也是直接对了一下笔试题，按照我改过的答案重新问了一遍，然后粗略的问了一些 Android 视图方面的问题，然后让我静待结果。没多久就回来对我说，很抱歉觉得我可能不是很适合他们公司。也在意料之中，表示没关系，也感谢他们的邀约，就回去了。为什么说这次面试是转折点呢，因为深圳其实小公司是非常非常多的，所以对于我这样一个毕业一年的人，有很多小公司看中了学历（虽然其实并不高）邀请去面试，在这之前所以虽然一直面试经验、简历含金量、知识水平都很不行，但毕竟面的是小公司，所以基本上只有工资谈不拢，没有直接拒绝的，这导致我一方面自我感觉良好觉得 Offer 很好拿，另一方面又很郁闷没有大 Offer。而这次面试真正打击到了我，完全撕开了我的舒适区，见识到了真正的面试应该是什么水平，以及自己懂的到底有多贫乏，至此之后，我开始一遍又一遍重写简历、总结面试、梳理知识点等等，直到现在完善出了一套自己的面试风格，也不再对面试感到紧张，甚至有时候还能反过来控制面试节奏，都要感谢这次面试带给我的挫折。一句话自勉一下：技术这条路崎岖坎坷，但我的征途，是星辰大海！

天地飞

  • 应聘部门：开发部

  • 应聘岗位：Android 开发工程师

  • 邀约时间：2018-08-20

  • 面试结果：入职

整体感受：人生中的第一份工作，虽然工作过程中有很多的不如意，但不论怎么说，毕竟是第一份工作，是毕业后第一个舞台，所以还是心存感恩的。这家公司的开发部，只有一个懂 Java 的人，因此也就只有一个懂 Android 的人，我去接前一个人的班，这唯一一个人就变成了我。面试基本没有什么过程可研，面试官也就是我入职后的直属 Leader，因为不懂这方面，所以没有问任何技术问题，基本上就是聊了一下公司的情况和岗位职责，就谈了工资。由于在本科期间错过了所有的校招，加上毕业后在广州和深圳找工作也不太顺利，甚至差点就参加培训机构了，所以对于找工作有点过于心急，当时收到天地飞的面试邀请，甚至直接定了最近一般的火车赶过去面试，为了省下开酒店的钱，半个小时不到面试完又赶回来，入职后也直接坐凌晨四点多的火车赶去上班。当时听到给的工资还是感觉不错的，但是因为没有经验，入职后才发现被坑了，所以也一直感觉有些遗憾。不过一个人负责这个岗位，对于应届毕业生来说，还是很有好处的。因为本科期间在学校做的东西，对于工作后的项目来说都太小儿科了，而工作项目有很明确的完成指标，这就被迫要自己去搞懂很多东西，甚至可以说是从 0 开始，所以在天地飞的工作经验，虽然没有做出含金量很高的项目，但却是极大地锻炼了我的个人能力，从这个角度出发，这份工作是值得的，毕竟锻炼的机会比工资对于一个新人来说更有用。而现在我也已经离职了，算是和公司友好分手，离职前也是尽心尽力把最后一份工作、最后一行代码、以及交接材料认真做好，也测试了无误才走的，就祝愿未来的我和天地飞，都能共同成长，共同进步吧！
